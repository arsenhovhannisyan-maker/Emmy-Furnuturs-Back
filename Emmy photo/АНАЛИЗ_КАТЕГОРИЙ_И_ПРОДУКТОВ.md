# Анализ папки «Emmy photo» — категории и продукты

## Структура

Папка **Emmy photo** — это каталог товаров Emmy Furnitures с **иерархией: категория → продукт → файлы**.

- **Уровень 1:** категория (например, «зеркала», «унитазы»).
- **Уровень 2:** продукт (название модели, например «Зеркало Эрика 60 Стандарт»).
- **Внутри продукта:** `Text Document.txt` (название, артикул, размеры, цены, описание) и фото (`.jpg`, `.png`).

## Содержимое папки продукта

В каждой папке продукта обычно есть:

1. **Text Document.txt**
   - Строка 1: **название продукта**
   - Строка 2: **артикул** (часто «нет»)
   - Строка 3: **бренд** (Emmy)
   - Таблица: **размеры и цены** (мм/см, цена в руб.)
   - Текст: **описание** (характеристики, комплектация)

2. **Изображения**
   - Фото товара (например `erika_70.jpg`, `7013.jpg`)
   - Схемы (например `*_shema.jpg`, `*_otkrytaya.jpg`)

## Выводы для импорта в БД / сидер

- **Категории:** 7 штук — можно завести в таблицу `categories` (название = имя папки категории).  
- **Продукты:** 104 штуки — привязать к категории по папке; название продукта = имя папки или первая строка `Text Document.txt`.  
- **Данные для продукта:** название, артикул, описание, размеры и цены — парсить из `Text Document.txt`.  
- **Фото:** пути к файлам `.jpg`/`.png` в папке продукта использовать для загрузки в хранилище и привязки к продукту в БД.

Если нужно, могу предложить структуру сидера (категории + продукты) или скрипт парсинга `Text Document.txt` под вашу БД.
